# AzureAIArchitect v2

> Upload an Azure architecture diagram. Get production-ready Bicep, Azure DevOps pipeline, compliance report, and cost analysis — all generated by a chain of real LLM-powered agents.

---

## 🧠 How it works — True Agentic Pipeline

Every step is performed by a dedicated AI agent with its own system prompt, Zod-validated output schema, and exponential back-off retry logic. Agents pass structured context to each other — no mocks, no hardcoded data.

```
Upload diagram
     │
     ▼
┌────────────────────┐
│  AnalyzerAgent       │  GPT-4o Vision → extracts resources,
│  (GPT-4o Vision)     │  dependencies, architecture pattern,
│                    │  security observations
└────────────────────┘
     │ DiagramAnalysis
     ▼
┌────────────────────┐
│  PolicyCompliance    │  GPT-4o → validates against
│  Agent              │  Default | HIPAA | PCI DSS | SOX
│                    │  returns violations + severity + fixes
└────────────────────┘
     │ ComplianceReport
     ▼
┌────────────────────┐
│  CostOptimization   │  GPT-4o → estimates Azure pricing,
│  Agent              │  right-sizing, reserved instances,
│                    │  immediate vs strategic savings
└────────────────────┘
     │ CostReport
     ▼
┌────────────────────┐
│  GeneratorAgent     │  GPT-4o → writes main.bicep,
│                    │  azure-pipeline.yml, parameters.json,
│                    │  README.md — with compliance fixes
│                    │  and cost opts baked in
└────────────────────┘
     │
     ▼
  📦 ZIP download
  (Bicep + pipeline + reports)
```

Progress is streamed to the UI in real time via WebSocket.

## ✨ What’s new in v2

| Area | v1 (old) | v2 (this branch) |
|---|---|---|
| LLM calls | Mocked / hardcoded | Real Azure OpenAI GPT-4o calls |
| Agent outputs | Plain objects | Zod-validated structured schemas |
| Error handling | Single try/catch | Exponential back-off retry (3x) |
| Agent context | None | Each agent receives upstream results |
| Compliance | Heuristic regex | LLM-reasoned per-resource analysis |
| Cost estimation | Placeholder $0 | LLM-estimated with pricing rationale |
| Status API | Hardcoded “completed” | Reads real trace store |
| Tests | None | Jest + mocked OpenAI for all agents |
| `package.json` name | `ai-superman` | `azure-ai-architect` |
| `.env.example` | Missing | Added with all required vars |
| Agent URLs | 5 fake env vars | Removed (agents are local classes) |

## 🚀 Quick Start

### Prerequisites
- Node.js 18+
- Azure OpenAI resource with GPT-4o deployment (or standard OpenAI API key)

### Setup

```bash
git clone https://github.com/dheeju25/AzureAIArchitect.git
cd AzureAIArchitect
git checkout feat/true-agentic-architecture
npm install
cp .env.example .env
# Edit .env — add AZURE_OPENAI_ENDPOINT and AZURE_OPENAI_API_KEY
npm run dev
```

Open http://localhost:3000

### Run tests

```bash
npm test
```

## 🛠️ Architecture

```
src/
├── agents/
│   ├── base.ts              # BaseAgent: LLM client, retry, Zod validation
│   ├── analyzer.ts          # GPT-4o Vision → DiagramAnalysis
│   ├── policyCompliance.ts  # GPT-4o → PolicyComplianceResult
│   ├── costOptimization.ts  # GPT-4o → CostOptimization
│   ├── generator.ts         # GPT-4o → Bicep + Pipeline + ZIP
│   └── orchestrator.ts      # Coordinates all agents, WebSocket progress
├── services/
│   ├── tracing.ts           # OpenTelemetry trace management
│   ├── traceQuery.ts        # Query trace store for live status
│   ├── telemetry.ts         # Azure Monitor initialisation
│   └── websocket.ts         # Socket.io real-time progress
├── types/
│   └── index.ts             # Canonical domain types
├── tests/
│   └── agents.test.ts       # Jest tests with mocked OpenAI
├── __mocks__/               # Test stubs for services
└── index.ts                 # Express app entry point
```

## 📦 Generated Output

Every processed diagram produces a ZIP containing:

| File | Description |
|------|-------------|
| `main.bicep` | Parameterised IaC with compliance fixes applied |
| `azure-pipeline.yml` | Multi-stage pipeline (validate → dev → prod with gates) |
| `parameters.json` | Tokenised environment parameters |
| `README.md` | Deployment guide |
| `reports/compliance-report.json` | Full violation list with remediations |
| `reports/cost-optimization.json` | Itemised cost breakdown + savings |
